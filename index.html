全景权限说明

FineReport 提供三种独立的权限体系，分别控制目录访问、URL 直接访问和设计器远程编辑。三者各自独立配置，互不影响。理解它们的作用范围和配置入口，是正确管理权限的前提。

目录权限

配置入口 权限管理 点击前往配置
除目录管理外权限管理下还包含人员管理、系统管理、数据连接、定时调度等权限配置
控制用户在决策平台目录里能否看到、编辑某个目录节点或报表
权限类型 查看权限、编辑权限、授权权限
适用范围 平台目录访问、应用中模板预览。用户必须拥有目录查看权限，才能在应用中预览该节点下的报表。
访问方式 决策平台目录打开（access + id 方式）
管理方式 超管或次管设置，支持按部门 / 角色 / 用户分配，支持继承与并集规则
其中模板展示支持的展示终端包括 PC、平板和手机。在设置模板时，可以选择需要的展示终端，以确保模板能够在相应的设备上正常显示

模板认证权限

配置入口 模板认证 点击前往配置
控制用户通过 URL 路径直接访问模板的权限，与目录权限相互独立
权限类型 分配模板查看权限
适用范围 设计器端或通过 viewlet= 路径直接访问模板。只有开启角色权限认证，才能精细控制用户访问的模板。
访问方式 decision/view/report?viewlet=xxx.cpt
管理方式 通常由超管设置，涉及模板的认证状态，影响预览与使用

远程设计权限

配置入口 需超管在远程设计中配置
控制用户通过设计器远程连接服务器进行报表编辑的权限
权限类型 远程设计器内文件夹及模板编辑权限
适用范围 仅适用于设计器端远程连接服务器，与平台访问权限完全独立
访问方式 设计器 → 服务器 → 远程设计连接
管理方式 仅超管可配置，无次级管理员；不支持用平台单点接口替代认证

⚠️ 常见误区

在权限管理点击前往配置里分配了目录权限，不代表用 URL 直接访问模板也有权限。两条路径由两套配置独立控制，需要分别配置。

如用户反馈"配了权限但 URL 访问报无权限"，请检查模板认证点击前往配置配置。
